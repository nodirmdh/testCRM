# AGENTS

## Общие принципы
- MVP-first: сначала минимально полезный продукт.
- Простая архитектура, без преждевременной оптимизации.
- Multi-tenant по умолчанию: в каждой сущности есть `organization_id`.
- Любые изменения требований отражаем в `docs/*.md`.
- Не добавляем зависимости без чёткой необходимости.
- Сначала план > потом реализация.
