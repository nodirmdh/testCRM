# Требования к продукту (Product Requirements)

## 1. Обзор продукта

Продукт: **Bendor LMS (рабочее название)** — SaaS‑платформа для управления учебными центрами, работающими оффлайн.

Целевая аудитория:

* Владельцы учебных центров
* Администраторы учебных центров

Модель монетизации:

* Ежемесячная подписка для учебных центров (B2B SaaS)

Формат обучения:

* Только оффлайн обучение
* Онлайн-уроки, видеоплатформа и чаты не входят в MVP

Ценность продукта:

* Упорядочивание данных (ученики, группы, финансы)
* Контроль доходов и расходов
* Прозрачность работы преподавателей
* Упрощение администрирования учебного центра

---

## 2. Роли пользователей

В системе предусмотрены только две роли:

* **Owner (Владелец)**
  Полный доступ ко всем разделам, включая финансы, подписку, управление пользователями.

* **Admin (Администратор)**
  Почти полный доступ: управление учениками, группами, преподавателями, расписанием, посещаемостью, оплатами, сертификатами.

> Разделения прав между Owner и Admin могут быть уточнены позже, но в MVP доступны одинаковые функции.

---

## 3. Основной функционал (MVP Scope)

### 3.1 Управление учебным центром

* Создание организации (учебного центра)
* Редактирование профиля центра: название, контакты, логотип
* Управление подпиской (активна / истекла)

---

### 3.2 Ученики

Функции:

* Добавление ученика
* Редактирование данных ученика
* Просмотр списка учеников
* Поиск и фильтрация

Данные ученика:

* ФИО
* Телефон (опционально)
* Адрес проживания
* Заметка (опционально)
* Статус: активный / неактивный

Родители (опекуны):

* Возможность добавить 1 или нескольких родителей
* Для каждого родителя: ФИО, телефон, тип родства

---

### 3.3 Преподаватели

Функции:

* Добавление преподавателя
* Редактирование данных
* Назначение преподавателя в одну или несколько групп

Данные преподавателя:

* ФИО
* Телефон
* Активность (работает / не работает)
* Комментарий (опционально)

---

### 3.4 Курсы и группы

Курсы:

* Название
* Описание (опционально)
* Длительность (опционально)

Группы:

* Название группы
* Привязка к курсу
* Преподаватель (может быть несколько)
* Расписание (текстовое поле на MVP)
* Список учеников
* Статус группы (активна / закрыта)

Функции:

* Создание и редактирование групп
* Запись ученика в группу
* Перевод ученика между группами
* Исключение ученика из группы

---

### 3.5 Посещаемость и занятия

Занятия:

* Создание занятия вручную или по кнопке "Провести занятие"
* Привязка к группе и преподавателю
* Дата проведения

Посещаемость:

* Отметка для каждого ученика: присутствовал / отсутствовал / опоздал
* Комментарий к занятию

Отчёты:

* Просмотр посещаемости по ученику
* Просмотр посещаемости по группе

---

### 3.6 Финансы — оплаты учеников

Система должна поддерживать:

* Назначение стоимости обучения для каждой группы (ежемесячно)
* Начисление оплаты каждому ученику за период (месяц)
* Фиксацию факта оплаты

Функции:

* Просмотр задолженностей по ученикам
* История оплат ученика
* Общий доход за период
* Доход по группе

Минимально поддерживаемые поля:

* Сумма
* Дата
* Способ оплаты (наличные / карта / перевод)
* Комментарий

---

### 3.7 Финансы — зарплаты преподавателей

Система должна поддерживать учёт зарплат преподавателей.

В MVP поддерживаются два типа расчёта:

* Фиксированная сумма за проведённое занятие
* Процент от дохода группы за период

Функции:

* Назначение правил расчёта зарплаты для преподавателя в группе
* Автоматический расчёт начислений за месяц
* Учёт факта выплаты зарплаты

Отчёты:

* Начислено преподавателю за период
* Выплачено
* Остаток долга

---

### 3.8 Сертификаты

Функции:

* Генерация сертификата для ученика
* Указание: ФИО ученика, курс, дата, уникальный номер
* Возможность скачать сертификат в PDF

Сертификат:

* Использует шаблон
* Имеет уникальный идентификатор

---

## 4. Аналитика и отчёты (MVP уровень)

Минимальные показатели:

* Общее количество учеников
* Количество активных групп
* Доход за месяц
* Доход по группам
* Задолженности учеников
* Зарплаты преподавателей за месяц

---

## 5. Ограничения MVP (что не входит)

В первую версию продукта **не входят**:

* Онлайн-уроки
* Видеоплатформа
* Чаты
* Мобильное приложение
* Публичный маркетплейс курсов
* Сложная система ролей
* Интеграции с внешними сервисами

Эти функции рассматриваются как возможное развитие продукта в будущем.

---

## 6. Нефункциональные требования

* Веб-приложение (доступ через браузер)
* Адаптивность для планшетов
* Авторизация по логину и паролю
* Безопасное хранение паролей
* Изоляция данных между организациями
* Производительность: система должна комфортно работать при 1 000+ учениках в одной организации

---

## 7. Критерии успеха MVP

MVP считается успешным, если:

* Учебный центр может полностью вести учёт учеников и групп в системе
* Владелец видит реальную картину доходов и расходов
* Администратор может работать без Excel
* Система используется в реальной работе минимум одним учебным центром
